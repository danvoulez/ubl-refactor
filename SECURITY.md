# Security Policy

## Reporting a vulnerability

Do not file public issues for security vulnerabilities.

- Report via email: `security@logline.foundation`.
- Report via GitHub Security Advisories (preferred if enabled).

Include:
- affected component/version
- reproduction steps or proof of concept
- impact assessment
- suggested mitigation (if known)

## Scope

This policy covers code and operational artifacts in this repository.

## Response targets

- Initial acknowledgment: within 5 business days.
- Triage and severity assessment: within 10 business days.
- Fix timeline: depends on severity and release risk; critical issues are prioritized.

## Disclosure

- Coordinated disclosure is preferred.
- Public disclosure should happen after a fix or mitigation is available.
