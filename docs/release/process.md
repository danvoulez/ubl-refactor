# Release Process

This process defines how maintainers cut a public release tag.

## 1) Prepare

- Confirm all required CI checks are green on `main`.
- Update `CHANGELOG.md` unreleased section.
- Verify docs entrypoints and release checklist are complete.

## 2) Version and tag

- Choose the next semantic version (`vMAJOR.MINOR.PATCH`).
- Create an annotated tag on the release commit.
- Push the tag to origin.

## 3) Publish

- Create GitHub release notes from tag.
- Link the release notes to `CHANGELOG.md` and compatibility policy.
- Attach attestations/artifacts if generated by CI.

## 4) Post-release

- Open next `Unreleased` section in `CHANGELOG.md`.
- Announce the release in repository channels.
- Track regressions as prioritized issues.
